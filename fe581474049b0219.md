# WebLink Hub

WebLink Hub 是一个面向技术研究人员、内容聚合者与信息分析师的轻量级外链资源归集与导航系统。该项目定位于将分散于互联网各处的技术文章、案例文档、参考手册等外部资源进行集中式收录、分类与快速检索，解决个人或团队在信息收集过程中产生的链接散落、查找困难、上下文缺失等问题。

WebLink Hub 不生产内容，而是通过结构化方式组织外部链接，为每个资源条目附加来源域名、资源类型、摘要标签与采集时间等元数据，形成可维护、可扩展的本地知识库骨架。适用于搭建个人书签管理后台、团队共享资源清单、技术周报素材库或自动化采集管道的前端展示层。

## 功能概览

- **链接入库与去重** 支持批量导入原始 URL，自动识别域名与路径结构，基于完整链接进行 MD5 哈希去重，避免重复收录。

- **元数据自动补全** 对每个收录链接尝试发起轻量级 HEAD 请求，获取响应状态码、内容类型与最后修改时间，补充基础网络元数据。

- **分类标签体系** 允许用户为每个资源定义一级分类（如后端架构、前端工程、运维监控、数据库理论）和自定义标签，支持多标签筛选与组合查询。

- **全文检索与过滤** 基于链接标题、来源域名、标签集合和摘要描述构建倒排索引，提供毫秒级关键词检索，同时支持按域名、按状态码等条件过滤。

- **数据导出与同步** 支持将收录的资源列表导出为 CSV、JSON 或 Markdown 表格格式，便于迁移至其他文档系统或进行离线分析。

- **收藏与阅读状态标记** 为每条记录提供星标收藏、已读/未读、重要程度（P0-P3）等状态字段，辅助个人阅读进度管理。

- **定期健康检查** 内置定时任务，对已收录链接进行可用性探测，自动标记失效链接（4xx/5xx）并生成异常报告。

- **RESTful API 接口** 提供基于 JSON 的完整 CRUD API，便于与其他自动化工具（如爬虫框架、CI/CD 流水线）集成。

## 应用场景

- **技术团队内部知识库沉淀** 研发团队在开发过程中会遇到大量外部参考链接（如 Stack Overflow 问答、官方文档片段、博客解决方案），通过 WebLink Hub 统一归档，并附加解决场景描述，方便后续同类问题快速检索。

- **个人技术阅读工作流管理** 技术爱好者每日浏览多个资讯站点与技术社区，可将值得深入阅读的文章链接存入系统，利用分类标签和阅读状态标记构建个人读库，避免信息过载与遗忘。

- **开源项目文档外链索引** 开源项目维护者需要收集相关的生态工具、插件列表、迁移案例等外部资源，使用 WebLink Hub 生成结构化的外链清单，随项目仓库一同发布，降低新用户的学习门槛。

- **自动化采集管道数据展示** 数据采集系统每日产出大量原始 URL，经清洗去重后存入 WebLink Hub，作为中间存储层，再通过其 API 对接前端看板或周报生成脚本，实现采集链路闭环。

- **技术活动与会议资料整理** 参与技术峰会或线下 meetup 后，将演讲 PPT 链接、视频回放地址、相关代码仓库等资料集中收录，按活动名称和演讲主题分类，方便会后回顾与分享。

## 快速开始

以下步骤帮助您在本地环境快速启动 WebLink Hub 服务。

```bash
# 1. 克隆代码仓库
git clone https://github.com/weblink-hub/weblink-hub.git
cd weblink-hub

# 2. 安装依赖（使用 Python 3.9+ 与 pip）
python -m venv venv
source venv/bin/activate  # Linux/macOS
# 或 .\venv\Scripts\activate  # Windows
pip install -r requirements.txt

# 3. 初始化数据库并运行开发服务器
python manage.py migrate
python manage.py runserver 0.0.0.0:8000
```

服务启动后，访问 http://localhost:8000 进入 Web 管理界面。默认管理员账号为 admin，初始密码在首次启动时打印于终端日志中。

## 常见问题

**Q: 导入大量链接时出现超时或内存占用过高怎么办？**

A: 系统默认单次导入上限为 1000 条，若超过此数量，建议使用脚本分批导入（参考 scripts/import_batch.py 实现）。对于元数据补全阶段，系统默认开启 5 个并发请求，可通过环境变量 `FETCHER_CONCURRENCY` 调整并发数以平衡性能与稳定性。同时，建议将数据库从 SQLite 切换至 PostgreSQL 以支持更高写入负载。

**Q: 健康检查任务对失效链接的判定标准是什么？**

A: 健康检查服务会对每个链接发起 GET 请求（跟随重定向），超时时间设为 10 秒。若返回状态码为 200 或 301/302（且最终可达），则标记为有效；若返回 4xx/5xx 或连接超时、DNS 解析失败等网络异常，则标记为失效。连续两次检查均失效的链接会进入「待确认」状态，第三次检查仍失效则自动标记为「已失效」，并记录失败原因。

**Q: 如何将现有浏览器书签批量迁移至 WebLink Hub？**

A: 大多数浏览器支持将书签导出为 HTML 文件（包含 `<a href="...">` 标签）。您可以使用项目提供的 `scripts/import_bookmarks.py` 脚本解析该 HTML 文件，提取所有链接及其标题，再通过 API 或管理后台的批量导入功能入库。具体操作步骤请参考 /docs/user-guide/ 中的「迁移指南」章节。
