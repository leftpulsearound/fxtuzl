# WebFront Archive

WebFront Archive 是一个面向前端技术研究者与全栈开发者的技术文章聚合与导航项目。本项目系统性地收集并整理了来自 kmvdvi 博客平台的历史技术文章，涵盖移动端开发、浏览器兼容性、HTTP 协议调试、前端工程化、UI 组件设计等多个技术领域。项目定位为技术资料的外链索引库，帮助开发者在碎片化的技术文章中快速定位到特定主题的实践案例与解决方案。

本项目适用于需要查阅历史技术文档、研究前端技术演进路径、或寻找特定问题实现参考的开发者。通过统一的资源索引机制，用户可以避免在海量技术博客中盲目检索，直接获取经过分类与归档的原始文章链接。


## 功能概览

- **多维度资源索引**：按文章编号与发布时间建立索引，支持快速检索特定技术主题的历史文章。

- **双端内容聚合**：同时收录 wap 端与 h5 端两类文章资源，覆盖移动端与桌面端两种访问场景。

- **原始链接直出**：所有资源链接均以原始格式呈现，不经过任何改写或重定向，确保访问路径的确定性。

- **结构化元数据标注**：每个资源条目关联文章编号与文件类型后缀，便于用户识别资源格式。

- **批量资源归档**：支持将大量历史文章链接整合为单一索引列表，适用于离线阅读与存档场景。

- **技术标签分类**：按文章 URL 中的编号段与路径特征，可辅助开发者识别文章所属的技术专题范围。

- **社区贡献扩展**：允许开发者通过 Pull Request 向索引库追加新的历史文章链接或修正失效链接。


## 应用场景

1. **前端技术演进研究**：研究人员可利用本索引库快速调取不同时期的前端技术文章，对比分析移动端与桌面端开发实践的变化轨迹。

2. **遗留系统维护参考**：开发者在维护旧项目时，可通过本库检索特定版本的技术实现细节，快速定位到与项目时间线匹配的历史文章。

3. **技术博客内容聚合**：技术博主或内容运营者可将本索引作为素材来源，批量获取原始文章链接用于二次整理或专题推荐。


## 快速开始

以下命令演示了如何将本项目的资源索引克隆至本地，并启动一个简单的静态服务用于浏览。

```bash
# 克隆项目仓库
git clone https://github.com/webfront-archive/webfront-archive.git

# 进入项目目录
cd webfront-archive

# 安装依赖（使用 npm 或 yarn）
npm install

# 启动本地开发服务器
npm run dev
```

执行上述命令后，可在本地浏览器中访问项目提供的资源导航页面，查看全部归档文章链接。

## 贡献指南

1. 在 GitHub 上 Fork 本仓库，并克隆到本地开发环境中。

2. 在 `resources/index.md` 文件中新增或修改资源链接，确保链接以原始格式原样添加，不添加额外协议前缀或路径改写。

3. 运行本地验证脚本 `npm run validate-links` 检查新增链接的格式规范性与可访问性。

4. 提交包含清晰变更说明的 Commit，并推送到个人 Fork 仓库。

5. 向本仓库的主分支发起 Pull Request，等待项目维护者审核与合并。


## 常见问题

**Q: 资源列表中的部分链接无法访问，应该如何处理？**

A: 由于本项目的资源列表均来自外部博客平台，部分历史链接可能因源站调整而失效。用户可通过提交 Issue 标记失效链接，或按照贡献指南提交 Pull Request 移除或替换确认失效的链接。项目维护者会定期执行链接验证脚本，并在主列表中标注失效链接的状态。

**Q: 是否可以添加本列表之外的其他技术文章链接？**

A: 可以。本项目欢迎开发者提交与前端技术、移动端开发、HTTP 协议、浏览器兼容性等主题相关的优质历史文章链接。新增链接需确保原始 URL 完整无误，并附带简要的主题标签说明。提交前请运行验证脚本确认链接可访问。

**Q: 项目是否会主动对资源链接进行内容备份或镜像？**

A: 本项目定位为外链索引库，不存储或备份源站的文章内容。所有资源链接均直接指向原始博客平台，用户访问时需依赖源站的可用性。建议用户在查阅重要资料时自行保存本地副本。
