# LinkVault Core

LinkVault Core 是一个面向开发者、技术研究人员与内容创作者的轻量级技术资源外链聚合与导航系统。该项目定位于将分散在互联网各处的优质技术文章、教程、文档与代码示例按结构化方式组织起来，解决个人开发者与团队在技术学习与问题排查过程中信息检索成本高、优质资源分散、收藏管理混乱等核心痛点。LinkVault Core 本身不生产内容，而是提供一套标准化的外链收录、分类、检索与展示框架，帮助用户高效建立自己的技术知识外链库。

## 功能概览

**结构化外链收录**：支持按技术领域、内容类型、来源站点等多维度对收录的 URL 进行分类与标注，便于后续检索与复用。

**全文元数据提取**：对收录的每一篇技术文章自动提取标题、发布时间、概要描述等元数据信息，形成标准化的卡片展示。

**标签化检索系统**：基于标签体系实现快速筛选，用户可按编程语言、框架版本、问题领域等标签定位所需资源。

**本地化缓存与快照**：对已收录的外链内容生成文本快照与本地缓存，在源站不可用时仍可查看核心内容。

**导入与导出机制**：支持批量导入 URL 列表，也支持将整个外链库导出为 JSON、Markdown 或 CSV 格式，便于迁移与分享。

**自定义分类视图**：允许用户根据自身关注的技术栈创建自定义分类视图，将不同目录下的外链聚合到同一看板。

**访问频率统计**：记录每条外链的点击与访问频次，自动生成高频资源推荐列表。

**RSS 订阅源生成**：为公开的外链库生成符合 RSS 2.0 规范的订阅源，便于他人订阅更新。

## 应用场景

技术团队内部知识库建设：技术团队可使用 LinkVault Core 搭建内部的技术文章导航站，将团队在开发过程中参考过的优秀外链统一收录，并按项目或技术方向分类，降低新成员的学习曲线。

个人开发者技术学习路径管理：个人开发者在学习新技术栈（如 Rust、Kubernetes、WebAssembly 等）时，可将阅读过的教程、API 参考、最佳实践文章统一收录，形成可追溯的学习路径，避免重复搜索。

技术社区内容聚合站：技术博客或社区运营者可使用 LinkVault Core 快速构建一个垂直领域的外链聚合页，将分散在各处的优质内容集中展示，为读者提供一站式的资源导航服务。

离线文档辅助工具：在内部网络受限或需要离线查阅的环境中，LinkVault Core 的本地缓存功能可作为文档辅助工具，帮助工程师在不连接外网的情况下快速查阅已收录的技术文章摘要与核心段落。

## 快速开始

以下命令演示了从 GitHub 克隆项目、安装依赖并启动开发服务器的完整流程。

```bash
# 克隆项目仓库
git clone https://github.com/linkvault/linkvault-core.git

# 进入项目目录
cd linkvault-core

# 安装依赖（使用 npm）
npm install

# 初始化本地数据库
npm run init-db

# 启动开发服务器（默认端口 3000）
npm run dev
```

## 安装要求

| 依赖 | 必需 | 说明 |
|------|------|------|
| Node.js 18.x 或更高版本 | 是 | 项目运行时环境，建议使用 LTS 版本 |
| npm 9.x 或更高版本 | 是 | 依赖管理与脚本执行工具 |
| SQLite 3 | 是 | 内嵌轻量级数据库，用于存储外链元数据与标签 |
| Git 2.30 或更高版本 | 是 | 用于版本控制与后续更新拉取 |
| 现代浏览器（Chrome 110+ / Firefox 110+ / Edge 110+） | 否 | 仅用于访问管理界面，后端 API 无浏览器要求 |
| 网络连接 | 否 | 仅首次启动时需连接互联网以下载依赖包，后续运行可完全离线 |

## 贡献指南

我们欢迎并鼓励社区贡献，无论是报告问题、提交补丁还是改进文档。请按照以下步骤参与贡献。

第一步：查阅问题列表与路线图。访问 GitHub Issues 页面查看当前已标记的待办任务、功能请求与已知缺陷，选择未被认领且与自身技能匹配的任务。

第二步：派生项目仓库并创建功能分支。从主仓库派生副本到个人账户下，克隆到本地后使用 `git checkout -b feature/描述性分支名` 创建独立分支，避免在主分支上直接修改。

## 常见问题

问：导入大量外链时如何处理重复 URL？

系统在导入过程中会自动对 URL 进行归一化处理（去除末尾斜杠、统一协议大小写等），并与数据库中已有记录进行比对。重复 URL 不会被重复插入，但会更新其最后检测时间与访问计数。用户可在导入配置中选择“跳过重复”或“覆盖已有元数据”两种策略。

问：本地缓存占用的存储空间如何管理？

缓存存储于 `./cache` 目录下，系统默认启用自动清理机制，当缓存总大小超过配置的 `cache.maxSize` 阈值（默认 500MB）时，会按最后访问时间（LRU）策略删除最久未使用的缓存条目。用户也可在管理界面手动清空全部缓存或针对特定域名进行清理。

问：能否在不启动 Web 界面的情况下仅使用命令行管理外链？

可以。LinkVault Core 提供了完整的 CLI 工具，支持通过命令行执行导入、导出、检索、统计等核心操作。在项目根目录下执行 `node cli.js --help` 可查看所有可用命令及其参数说明。该模式适用于自动化脚本与无图形界面的服务器环境。
