# TechLink Aggregator

TechLink Aggregator 是一个面向开发者与技术研究人员的开源技术资源聚合平台，专注于收集、整理和索引来自互联网的高质量技术文章、教程与工程实践案例。该项目并非一个传统的博客系统或内容管理系统，而是一个结构化的外链资源导航工具，帮助技术团队在信息过载的环境中快速定位到有价值的原始内容。

项目定位为技术团队的辅助知识库，适用于需要持续跟踪技术动态、收集技术资料或建立内部技术文档索引的场景。通过统一的数据采集与分类机制，TechLink Aggregator 将分散在多个来源的技术内容以统一的条目格式呈现，并提供基础的检索与分类浏览功能。本项目不存储文章原文，仅保留链接、摘要与元数据，所有版权归属原始作者与站点。

## 功能概览

**结构化资源收录**：每个资源条目包含标题、原始链接、来源域名、采集时间与内容摘要，所有数据以 Markdown 表格形式存储在本地仓库中，便于版本管理与协作编辑。

**多维度分类索引**：资源按技术领域、应用场景与难度等级进行标签化分类，支持通过目录树快速浏览某一类别的所有资源。

**全量链接校验**：内置链接有效性检查脚本，定期扫描资源列表中的 URL 状态，自动标记失效链接并生成报告。

**静态站点生成适配**：项目数据结构兼容主流静态站点生成器，可一键导出为 HTML 文档，部署为内部技术导航页。

**命令行交互工具**：提供 Python 编写的 CLI 工具，支持新增资源、批量导入、关键词检索与统计信息查看。

**轻量级依赖**：项目仅依赖 Python 3.8+ 与标准库，无需数据库或外部服务即可完整运行。

## 应用场景

技术团队内部知识库建设：团队技术负责人可以将本项目作为团队周报或技术分享的素材池，定期收录团队关注的技术博客与工程案例，形成可追溯的技术决策参考依据。

个人技术阅读清单管理：开发者可以 fork 本项目，将日常阅读的技术文章以资源条目的方式保存下来，通过标签分类构建个人技术成长路线图。

技术社区内容聚合：开源社区维护者可以使用本项目收集与项目相关的第三方教程、使用案例与生态工具，降低新贡献者的信息获取成本。

技术文档写作素材库：技术作者在撰写技术方案或白皮书时，可通过本项目的分类索引快速找到相关领域的权威资料与引用来源。

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/techlink-aggregator/techlink-aggregator.git
cd techlink-aggregator

# 安装依赖（Python 3.8+）
pip install -r requirements.txt

# 运行资源索引构建
python cli.py build --output ./output

# 启动本地预览服务
python cli.py serve --port 8080
```

## 贡献指南

新增资源条目：通过 CLI 工具提交新资源，命令为 `python cli.py add --url <原始链接> --title <标题> --tags <标签列表>`，工具会自动采集元数据并生成条目文件。提交前请确认资源内容与已有条目不重复，且原始站点允许外链引用。

更新分类体系：如需新增或修改分类标签，请编辑 `data/taxonomy.yaml` 文件，并确保所有已有条目的标签与更新后的体系保持一致。修改后运行 `python cli.py validate` 进行完整性校验。

链接有效性维护：定期运行 `python scripts/check_links.py` 检查资源列表中的链接状态。对于返回 4xx 或 5xx 状态的链接，请在条目文件中标记 `[broken]` 并注明检测日期，便于后续清理或替换。

文档改进：欢迎对 README、用户手册或架构说明提出改进。文档采用 Markdown 格式，修改后请确保排版一致且无拼写错误。重大文档变更建议附带变更说明。

提交代码或数据变更时，请使用常规提交规范，提交信息格式为 `<类型>: <简短描述>`，类型包括 `feat`、`fix`、`docs`、`chore`、`data` 等。提交前请运行测试套件确保现有功能不被破坏。

## 常见问题

问：资源条目中的原始链接如果失效了怎么办？

答：项目内置的链接检查脚本会标记失效链接。用户可以在条目文件中保留链接作为历史记录，并在备注中注明失效日期。如果原始站点有迁移或替代链接，可以更新条目中的 URL 字段并添加备注说明。对于长期失效且无法找到替代来源的条目，可以将其移入 `data/archived/` 目录。

问：本项目与普通书签管理器有什么区别？

答：本项目侧重于结构化、可协作的资源索引，而非个人书签的实时同步。每个资源条目除了链接外，还包含采集时间、分类标签、内容摘要等元数据，支持版本管理和批量操作。项目本身是一个 Git 仓库，适合团队共同维护，且不依赖任何第三方在线服务。

问：如何将本项目部署为团队内部可访问的导航站点？

答：使用 `python cli.py build` 命令生成静态 HTML 文件，将 `output/` 目录下的所有文件上传至任意静态托管服务即可。推荐使用 Nginx、Caddy 或 GitHub Pages 进行部署。如需访问控制，可在 Nginx 层面配置基础认证或 IP 白名单。
