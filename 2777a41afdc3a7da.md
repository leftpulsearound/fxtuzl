# WebArchive Navigator

WebArchive Navigator 是一个面向技术研究、历史内容回溯与互联网资源索引的轻量级外链汇总与导航系统。该项目定位于帮助开发者、技术文档撰写者、数据归档人员以及内容研究人员，快速定位并访问分布于各类技术博客、移动端文章平台的历史技术资料与讨论内容。

项目本身不存储任何原始文章内容，仅提供结构化的外部链接索引与分类导航能力。通过对大量分散于移动端技术博客平台的文章链接进行归类、标签化与状态监控，WebArchive Navigator 能够显著降低技术资料查找与筛选的时间成本，尤其适用于需要批量查阅特定领域历史技术文档、故障排查记录或版本演进讨论的场景。

---

## 功能概览

- **外链索引管理**：提供对大量外部技术文章链接的集中式存储、分类与检索能力，支持按文章ID、来源域名、大致发布时间等维度进行筛选与排序。

- **链接可用性监控**：内置定时检测机制，定期对收录的外链进行HTTP状态检查，标记失效链接、重定向链接及响应异常链接，帮助维护索引库的健康度。

- **分类与标签系统**：支持为每一条外链赋予多个分类标签，例如“前端开发”、“运维故障”、“数据库调优”、“安全漏洞”等，便于按技术领域快速筛选。

- **批量导入与导出**：支持通过CSV、JSON或纯文本列表批量导入外部链接，并支持将当前索引库导出为标准格式，便于迁移或与其他工具集成。

- **搜索与过滤**：提供基于文章标题关键词、URL片段、域名、状态码等多条件的组合搜索功能，支持正则表达式过滤，满足精细化的检索需求。

- **访问统计与热度排序**：记录每条外链的点击次数与最后访问时间，支持按热度、更新时间或添加时间进行排序，帮助识别高频参考资源。

- **RSS订阅与变更通知**：支持生成基于标签或分类的RSS订阅源，当索引库中新增或更新特定分类的外链时，订阅者可以及时获得通知。

- **开放API接口**：提供RESTful API，允许第三方工具或脚本以编程方式查询、添加或更新外链记录，便于嵌入更复杂的自动化工作流。

## 应用场景

- **技术文档撰写参考素材收集**：技术文档工程师或开源项目维护者在撰写版本发布说明、迁移指南或故障排查手册时，可通过WebArchive Navigator快速检索历史讨论链接，引用可靠的外部资料作为补充说明。

- **历史故障案例复盘**：运维团队或SRE工程师在处理重复性故障时，可以利用本系统快速找到以往相同或相似故障的外部讨论链接，参考当时的解决思路与社区反馈，缩短故障恢复时间。

- **技术调研与竞品分析**：技术选型团队或架构师在进行技术调研时，可通过本系统批量查阅特定技术栈或中间件在移动端技术社区的历史讨论热度与问题演变趋势，辅助决策。

- **开源项目外部资源整合**：开源项目维护者可以将项目相关的技术博客、使用案例、社区问答等外部资源统一收录至WebArchive Navigator，作为项目Wiki或文档站的外部延伸，方便贡献者与用户查阅。

## 快速开始

以下指令演示了如何从GitHub克隆项目仓库、安装依赖并启动开发服务。

```bash
# 克隆项目仓库
git clone https://github.com/your-org/webarchive-navigator.git

# 进入项目目录
cd webarchive-navigator

# 安装项目依赖（使用npm）
npm install

# 初始化数据库（首次运行）
npm run init-db

# 启动开发服务器（默认端口3000）
npm run dev
```

启动成功后，访问控制台输出的本地地址即可开始使用Web界面进行外链管理。

## 常见问题

**问：WebArchive Navigator是否存储外部文章的内容副本？**

答：不存储。本系统仅索引并存储外部链接的URL、标题、标签、添加时间等元数据，不缓存或复制任何外部文章的内容。所有内容访问均需跳转至原始来源网站，项目本身不承担原始内容的版权责任。

**问：链接可用性检测的频率是多少？是否可配置？**

答：默认情况下，系统每24小时对所有收录的外链执行一次HTTP状态检测。检测频率、超时时间、重试次数等参数均可在 `.env` 环境变量文件中配置，支持按标签或按链接分组单独设定检测策略，以平衡系统资源消耗与监控实时性需求。

**问：如何迁移或备份我所维护的外链索引库？**

答：您可以使用项目内置的导出脚本（`scripts/export-json.js`）将整个索引库导出为JSON文件，该文件包含所有链接的元数据与标签信息。导入时使用 `scripts/import-csv.js` 或通过API批量上传即可完成迁移。建议定期执行导出备份，并将备份文件存储于外部持久化存储服务中。
