# ResourceBridge

ResourceBridge 是一个面向技术社区的开源外链资源汇总与导航系统。该项目定位于解决技术文档、博客文章、教程视频等优质外部内容散落在互联网各处、难以集中发现与检索的问题，帮助开发者、技术写作人员与社区运营者快速建立结构化的知识索引。ResourceBridge 不生产内容，而是通过精心维护的链接目录，将高价值技术资源以可检索、可分类、可版本化的方式组织起来，降低信息摩擦，提升学习与研发效率。

本项目面向以下目标用户：需要持续追踪特定技术领域动态的工程师、负责技术社区内容运营的编辑团队、撰写技术博客或教程的独立作者，以及希望构建内部技术知识库的企业研发部门。ResourceBridge 通过提供标准化的资源描述格式、分类标签体系和自动化检查工具，使大规模外链管理变得可靠且低成本。

## 功能概览

**结构化资源目录** 提供多级分类与标签系统，支持按技术领域、内容类型、难度等级等维度筛选资源，每个资源条目包含标题、摘要、原始链接、收录时间与维护状态。

**自动化链接健康检查** 内置链接有效性检测工具，可定时扫描已收录的 URL，识别 404、超时、重定向等异常状态，并生成报告供管理员处理。

**全文检索与过滤** 基于资源标题、描述和标签构建的轻量级索引，支持布尔查询和模糊匹配，帮助用户在海量链接中快速定位所需内容。

**资源导入与导出** 支持批量导入 URL 列表（支持 TXT、CSV、Markdown 格式），并可导出为 JSON、YAML 或 HTML 书签文件，便于与其他工具集成。

**版本化变更追踪** 每次增删改操作均记录操作人、时间与变更说明，支持回溯历史版本，确保资源目录的审计合规性。

**社区协作工作流** 集成 Issue 提交与 Pull Request 模板，允许外部贡献者推荐新资源、报告失效链接或更新信息，维护者可通过统一后台审核与合并。

**自定义分类模板** 允许用户根据自身需求创建分类模板，例如按编程语言、框架版本、云服务商等维度定义专属分类树，并共享给团队使用。

**访问统计与热度分析** 记录每个资源链接的点击次数和最近访问时间，生成热度排行，辅助判断资源实际价值，便于定期清理低质量条目。

## 应用场景

**技术团队内部知识库构建** 研发团队可将日常参考的官方文档、开源项目地址、内部工具链接、故障排查笔记等统一纳入 ResourceBridge 管理，替代散乱的浏览器书签，新成员入职时可快速获取完整技术地图。

**技术社区内容聚合与推荐** 社区运营者可使用 ResourceBridge 整理每周优质文章、视频教程和工具推荐，生成可公开访问的资源导航页，提升社区内容复用率，减少重复回答基础问题的时间。

**个人学习路线规划与跟踪** 自学者可围绕特定技术栈（如 Rust 后端开发、机器学习工程化）建立专属资源清单，通过版本化记录追踪自己的学习轨迹，并定期清理已掌握内容的过时链接。

**技术文档撰写素材库** 技术博主与文档作者可将参考文献、数据来源、代码示例出处统一存于 ResourceBridge，在撰写长文时快速引用，确保引用链路的完整性与可追溯性。

## 快速开始

以下命令将 ResourceBridge 克隆至本地，安装依赖并启动开发服务器。

```bash
git clone https://github.com/resourcebridge/resourcebridge.git
cd resourcebridge
npm install
npm run dev
```

执行完上述步骤后，打开浏览器访问 http://localhost:3000 即可使用本地实例。生产环境部署请参考 `docs/deployment.md` 文档。

## 安装要求

| 依赖组件 | 必需版本 | 说明 |
|---------|---------|------|
| Node.js | >= 18.0.0 | 运行时环境，推荐使用 LTS 版本 |
| npm | >= 9.0.0 | 包管理器，用于安装项目依赖 |
| SQLite | >= 3.35.0 | 内置数据库，用于存储资源条目与元数据，无需额外安装 |
| Git | >= 2.30.0 | 版本控制工具，用于克隆仓库和提交变更 |
| curl | >= 7.68.0 | 用于链接健康检查模块的 HTTP 请求，Linux/macOS 默认已安装 |
| cron | 系统自带 | 定时任务调度器（可选），用于自动化链接扫描 |
| Docker | >= 20.10.0 | 容器化部署（可选），如需使用 Docker 镜像部署方式 |
| Python | >= 3.9.0 | 仅当启用数据分析扩展时需要，用于生成访问统计图表 |

## 贡献指南

ResourceBridge 欢迎社区贡献，无论是推荐新资源、报告失效链接、提交代码修复还是完善文档，均对项目有实质帮助。请遵循以下步骤参与贡献。

第一步：查阅现有 Issue 与 Pull Request。在提交新贡献前，请先浏览 GitHub Issues 和 Pull Requests 列表，确认没有重复或正在进行的相关工作。如发现已有相似议题，可在该议题下补充信息或表达支持。

第二步：Fork 仓库并创建功能分支。从主仓库 Fork 副本到个人账户，然后基于 main 分支创建描述性的分支名称，例如 `feat/add-python-category` 或 `fix/link-checker-timeout`。

第三步：遵循代码规范与测试要求。提交代码前请运行 `npm run lint` 和 `npm run test` 确保风格一致且所有单元测试通过。新增功能需附带相应的测试用例，修改现有功能需更新相关测试。

第四步：提交变更并撰写清晰 Commit Message。Commit Message 请使用约定式提交格式（Conventional Commits），如 `feat: 添加按标签过滤资源的 API` 或 `docs: 更新部署文档中的环境变量说明`。确保每个 Commit 逻辑独立且可审阅。

第五步：发起 Pull Request 并参与讨论。从功能分支向主仓库的 main 分支发起 Pull Request，填写 PR 模板中的检查清单，描述变更目的与影响范围。维护者会在 48 小时内进行初审，并可能提出修改意见，请保持关注并及时响应。

## 常见问题

**问：如何批量导入大量 URL？是否支持自动获取标题和摘要？**

ResourceBridge 支持通过 TXT、CSV 或 Markdown 列表格式批量导入 URL。导入时，系统会尝试自动抓取每个 URL 的页面标题和 meta 描述作为默认摘要，但受限于目标网站的 robots 协议和响应速度，建议导入后手动审核和补充关键字段。对于大规模导入（超过 1000 条），推荐使用 CLI 脚本 `scripts/bulk-import.js` 并配合后台队列处理。

**问：链接健康检查的检测频率是多少？误报如何处理？**

默认情况下，系统每 24 小时执行一次全量链接扫描，并在每次资源被点击时触发实时检测。若检测到异常状态（如 4xx 或 5xx），会标记为“待确认”并记录状态码。由于部分网站存在临时维护或反爬机制，可能存在误报，管理员可在后台手动重新检测或忽略特定 URL 的异常状态，系统会根据忽略列表跳过后续扫描。

**问：能否将 ResourceBridge 部署到内网环境，不连接外网？**

可以。ResourceBridge 的核心功能（资源管理、检索、分类、导出）完全依赖于本地 SQLite 数据库，不需要访问外网。仅链接健康检查模块需要向外发起 HTTP 请求，您可在配置文件中将 `healthCheck.enabled` 设为 `false` 以禁用该功能，或配置代理服务器。Docker 部署时同样支持离线模式，只需在构建镜像时提前下载所有依赖包即可。
