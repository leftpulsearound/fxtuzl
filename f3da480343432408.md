# WebLink Hub

WebLink Hub 是一个面向开发者、技术研究人员与内容策展人的高密度技术文章外链聚合平台。本项目定位于系统化收录来自特定技术博客站点的深度技术文章，为使用者提供可检索、可分类、可快速访问的参考链接集合。项目本身不存储文章内容，仅作为导航索引存在，帮助目标用户从大量分散的链接中快速定位可能感兴趣的技术话题。

本项目面向的典型用户包括：需要跟踪特定博客站点更新动态的订阅者、进行技术专题调研时需批量查阅资料的工程师、以及希望将外链资源整合进自身知识库的内容管理者。通过将同一来源的数百篇文章链接进行集中整理与分类展示，项目有效降低了用户在多个页面间反复跳转查找的时间成本。

## 功能概览

**按文章ID索引** 项目以文章唯一标识符为核心组织维度，每个链接均附带原始文章ID，便于用户快速记忆与回溯。

**多层级分类导航** 基于文章元数据信息，将链接按技术领域、写作时间或内容形式进行逻辑分组，提供树状分类浏览入口。

**原始链接直出** 所有链接均保留博客站点的原始URL结构，不添加任何中间跳转或追踪参数，确保访问路径的透明性与可预期性。

**轻量级本地部署** 项目以静态HTML与纯文本格式为主，无需数据库或后端服务，克隆后即可在本地浏览器中打开使用。

**可扩展的链接数据库** 链接清单采用结构化数据格式（JSON/CSV）存储，支持用户通过脚本批量增删条目或进行字段扩展。

**快速模糊检索** 集成前端检索脚本，允许用户基于文章ID或URL片段对当前链接列表进行实时筛选。

**定期同步机制** 提供可配置的同步脚本模板，支持用户设定定时任务，从源站点自动获取新增文章链接并更新本地索引。

**访问状态标记** 对每条链接可手动标注访问状态（待查阅/已读/需精读），辅助个人阅读进度管理。

## 应用场景

技术日报快速浏览 用户每日早晨打开本地部署的索引页面，按分类浏览新增或未读链接，在30分钟内完成对数十篇潜在技术文章的标题与摘要扫描，筛选出当日值得精读的内容。

专题调研资料汇集 当进行某项技术专题（如前端性能优化、分布式系统一致性）研究时，用户可通过项目内置的分类标签或全文检索功能，从数百条历史链接中快速提取与该专题相关的文章，形成定制化的阅读清单。

知识库外链整合 知识管理爱好者可将本项目作为外部引用源，在使用Obsidian、Notion等工具构建个人知识库时，通过脚本将本项目中的链接批量导入，并关联到对应的知识节点中。

团队技术分享池 技术团队可内部部署本项目，成员定期向其中新增有价值的文章链接，并利用访问状态标记功能标注推荐等级，使团队的技术视野积累变得可量化、可追溯。

## 快速开始

以下步骤帮助您在本地环境中完成项目的克隆、依赖安装与启动运行。

```bash
# 克隆项目仓库至本地
git clone https://github.com/your-org/weblink-hub.git

# 进入项目根目录
cd weblink-hub

# 安装项目依赖（若项目包含检索脚本或同步工具，基于Node.js环境）
npm install

# 启动本地开发服务器（默认占用端口8080）
npm run serve
```

执行上述命令后，在浏览器中访问 http://localhost:8080 即可看到项目主界面。若仅需静态浏览功能，可直接双击项目根目录下的 index.html 文件在浏览器中打开，无需执行安装与启动步骤。

## 安装要求

本项目在运行前需确保本地环境满足以下依赖条件。若仅使用静态HTML浏览模式，则无需安装任何额外软件，仅需一个现代浏览器即可。

| 依赖名称 | 必需性 | 说明 |
|---------|-------|------|
| Node.js (v16.x 或更高) | 可选 | 仅当需要运行同步脚本或检索服务时必需，纯浏览模式无需安装 |
| npm (v8.x 或更高) | 可选 | 随Node.js一同安装，用于管理项目工具链依赖 |
| 现代浏览器（Chrome/Firefox/Edge最新两个版本） | 必需 | 用于渲染界面与执行前端检索逻辑，建议启用JavaScript |
| 互联网连接 | 必需 | 首次加载及后续访问时，用于解析原始链接指向的文章内容 |
| 文本编辑器（VSCode/Sublime等） | 可选 | 若用户希望编辑链接数据库或修改分类配置，建议准备 |

## 项目结构

```
weblink-hub/
├── index.html                     # 主入口页面，提供分类导航与检索界面
├── about.html                     # 项目说明页，包含版本信息与使用声明
├── README.md                      # 项目自述文件（本文档）
├── LICENSE                        # MIT许可证全文
├── package.json                   # npm项目配置文件，定义依赖与脚本命令
├── package-lock.json              # 依赖版本锁定文件
├── .gitignore                     # Git版本控制忽略规则配置
├── config/
│   ├── categories.json            # 文章分类映射配置，定义分类名称与ID关联
│   └── sync-config.json           # 同步任务参数配置，含源站地址与更新频率
├── data/
│   ├── links.db.json              # 主要链接数据库，存储全部文章URL及元数据
│   ├── links.archive.json         # 历史归档版本，用于回滚与差异比对
│   └── tags.index.json            # 标签索引，支持多标签交叉检索
├── scripts/
│   ├── sync.js                    # 同步脚本，从源站抓取新增链接并更新数据库
│   ├── dedup.js                   # 去重工具，扫描数据库合并重复条目
│   ├── export-csv.js              # 导出工具，将链接数据库转为CSV格式
│   └── validate-urls.js           # 链接有效性检查工具，标记无法访问的条目
```

## 贡献指南

本项目欢迎外部贡献者参与链接数据库扩充、功能优化与文档完善。请遵循以下步骤提交贡献。

第一，Fork本仓库至个人账户，并克隆到本地开发环境。在本地新建一个功能分支，分支命名格式为 feature/简述变更内容 或 fix/简述修复问题，避免在主分支上直接修改。

第二，若为新增或修改链接数据，请严格按照 /docs/data-format.md 中定义的JSON Schema编辑 links.db.json 文件，确保新增字段不与现有结构冲突。若为代码功能改进，请在提交前运行 npm test 确保所有现有测试用例通过，并为新增功能编写对应的测试文件放置在 /tests 目录下。

第三，提交变更时使用语义化提交信息格式，例如 feat: 增加按文章发布时间排序功能 或 fix: 修复暗色模式下分类标签显示异常。提交信息需使用英文或中文，但需保持全篇一致性。

第四，向本仓库的 main 分支发起Pull Request，在PR描述中清晰说明变更目的、实现方式及影响范围。若变更涉及数据文件，请附带说明新增链接的来源与分类依据。项目维护者将在三个工作日内进行审查与合并。

## 常见问题

问：项目是否需要始终保持网络连接才能使用？

答：项目本身在本地加载时不需要网络连接，所有链接数据均存储在本地 JSON 文件中。但用户点击任意链接访问原始文章时，需要设备具备互联网连接能力，因为链接指向的是外部博客站点。若需离线查阅，建议用户自行将感兴趣的文章内容保存至本地。

问：如何更新链接数据库以获取最新文章？

答：项目提供了两种更新方式。手动更新：运行 npm run sync 命令，脚本会基于 config/sync-config.json 中的配置从源站抓取最新文章列表并与本地数据合并。自动更新：在操作系统任务计划程序（Windows）或 cron（Linux/macOS）中配置定时执行 npm run sync 命令，建议设置间隔为每日一次。同步过程中会自动跳过已存在的文章ID，不会产生重复条目。

问：检索结果不准确或遗漏部分链接应如何处理？

答：前端检索功能基于文章ID和URL路径中的关键词进行匹配，不涉及文章正文内容。若发现检索遗漏，请首先检查 /data/links.db.json 文件中是否确实包含目标链接。若存在但未被检索到，可尝试修改 /scripts/search.js 中的匹配正则表达式或模糊度阈值参数。对于更复杂的检索需求，建议将数据导出为CSV后使用外部数据分析工具处理。
