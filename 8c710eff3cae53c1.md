# WebLink Hub

WebLink Hub 是一个面向技术开发者与系统架构师的轻量级外链资源聚合与导航平台。该项目定位于将分散在各类技术博客、官方文档、社区讨论中的高质量外部链接进行结构化收集、分类归档与快速检索，帮助技术团队和个人在信息过载的环境中高效定位所需的开发资源、运维案例与架构参考。

WebLink Hub 并非一个传统的搜索引擎或书签管理器，而是一个专注于技术内容外链的精选集。其核心目标用户包括后端开发工程师、DevOps 实践者、技术决策者以及开源项目维护者。通过该项目，用户能够以极低的成本搭建一个内部或公开的技术资源导航站，将日常调研、故障排查、技术选型过程中积累的优质链接进行集中管理，并借助项目的分类体系与元数据标注，实现团队内部的知识沉淀与共享。

## 功能概览

**批量外链导入与解析** 支持从文本文件、CSV 表格或直接粘贴的 URL 列表中批量导入链接，系统自动识别 URL 格式并提取基础元数据。

**自定义多级分类体系** 允许用户根据技术栈、应用场景或业务领域创建无限层级的分类目录，每个链接可归属至多个分类标签。

**全文检索与过滤** 基于链接标题、描述、来源域名及分类标签提供快速全文搜索，同时支持按日期、访问量、分类进行组合过滤。

**链接健康状态监控** 定时对已收录链接进行可用性探测，自动标记失效或响应超时的链接，并生成状态报表。

**访问统计与热度分析** 记录每个外链的点击次数、最后访问时间与来源 IP 聚合，提供简单的热度排序与趋势展示。

**Markdown 格式数据导出** 支持将当前分类下的所有链接导出为标准 Markdown 列表格式，便于嵌入技术文档、项目 README 或 Wiki 页面。

**团队协作权限控制** 提供基于角色的访问控制，区分管理员、编辑者与只读访客三种权限级别，适用于团队内部知识库场景。

**开放 API 接口** 提供 RESTful API 用于链接的增删改查及分类管理，便于与其他内部系统或自动化脚本进行集成。

## 应用场景

技术团队内部知识库建设：开发团队在日常工作中会积累大量有价值的参考链接，包括第三方库文档、性能调优案例、故障复盘报告等。WebLink Hub 可作为团队内部的技术收藏夹，将散落在个人浏览器收藏夹或即时通讯群组中的链接统一收归，并按照微服务、数据库、前端框架等维度进行分类，新成员入职时可快速通过该平台了解团队常用的技术栈与参考资源。

技术选型与调研辅助：架构师在进行技术选型时，需要对比多个候选方案的特点、性能数据与社区活跃度。WebLink Hub 的链接分类与热度统计功能可帮助架构师将收集到的对比文章、官方性能测试报告、社区讨论帖集中管理，形成结构化的调研材料，支撑最终的技术决策。

开源项目文档外链管理：开源项目的 README 或官方文档中通常需要引用大量外部资源，如依赖库主页、协议说明、参考实现等。维护者可使用 WebLink Hub 统一管理这些外链，并利用 Markdown 导出功能快速生成格式规范的链接列表，避免手动维护 README 时出现的链接格式错误或失效问题。

个人技术写作素材库：技术博主或文档写作者在撰写文章时，需要引用大量外部资料作为论据支撑。WebLink Hub 可作为写作素材的临时仓储，按主题分类存放待引用的文章链接，结合搜索功能快速调取，提高写作效率。

## 快速开始

以下操作步骤演示了如何在本地环境中快速启动 WebLink Hub 服务。

```bash
# 克隆项目仓库至本地
git clone https://github.com/weblink-hub/weblink-hub.git

# 进入项目根目录
cd weblink-hub

# 安装项目依赖（使用 npm 或 yarn）
npm install

# 复制环境变量配置文件模板
cp .env.example .env

# 编辑 .env 文件，配置数据库连接字符串与端口参数
# 默认使用 SQLite 作为嵌入式数据库，无需额外安装

# 执行数据库迁移，初始化数据表结构
npm run migrate

# 启动开发服务器，默认监听 3000 端口
npm run dev
```

服务启动后，在浏览器中访问 `http://localhost:3000` 即可进入 WebLink Hub 的管理界面。首次访问将引导创建管理员账户。

## 安装要求

WebLink Hub 采用 Node.js 作为运行时环境，使用 SQLite 作为默认数据库，亦可切换至 PostgreSQL 或 MySQL 用于生产部署。以下为必需的依赖项与推荐版本。

| 依赖组件 | 必需性 | 说明与版本建议 |
|---------|--------|--------------|
| Node.js | 必需 | 版本 18.0.0 或更高，推荐使用最新的 LTS 版本 |
| npm 或 yarn | 必需 | npm 8.0.0 以上或 yarn 1.22.0 以上，用于包管理 |
| SQLite 3 | 必需（默认） | 内置依赖，无需单独安装，用于开发与小型部署 |
| PostgreSQL 14+ | 可选 | 生产环境推荐，需单独安装并配置连接 |
| Redis 6.0+ | 可选 | 用于会话存储与缓存加速，提升高并发下的响应性能 |
| Nginx 或 Apache | 可选 | 反向代理配置，用于静态资源缓存与负载均衡 |

## 项目结构

WebLink Hub 采用模块化的项目结构，将核心业务逻辑、数据访问层、用户界面与基础设施配置进行了清晰分离。

```
weblink-hub/
├── src/                                  # 源代码主目录
│   ├── api/                              # RESTful API 路由定义
│   │   ├── v1/                           # API 版本 1 的路由
│   │   │   ├── links.js                  # 链接资源的 CRUD 操作端点
│   │   │   ├── categories.js             # 分类管理端点
│   │   │   ├── users.js                  # 用户认证与权限管理端点
│   │   │   └── health.js                 # 服务健康检查端点
│   │   └── middleware/                   # 请求中间件
│   │       ├── auth.js                   # JWT 令牌验证中间件
│   │       └── rate-limit.js             # 请求频率限制中间件
│   ├── core/                             # 核心业务逻辑层
```

## 常见问题

问题：如何从 SQLite 迁移至 PostgreSQL 作为生产数据库？

回答：首先在 .env 文件中修改 DATABASE_URL 环境变量，将其指向 PostgreSQL 的连接字符串，格式为 `postgresql://用户名:密码@主机:端口/数据库名`。然后运行 `npm run migrate` 命令，系统将自动识别数据库类型并在 PostgreSQL 中创建相应的表结构。原有的 SQLite 数据不会自动迁移，若需要保留历史数据，可使用项目提供的 `scripts/export-db.js` 和 `scripts/import-db.js` 工具进行手动导出与导入。

问题：链接健康检查的探测频率如何调整？检测结果是否会影响前端展示？

回答：健康检查的频率由环境变量 `HEALTH_CHECK_INTERVAL` 控制，单位为分钟，默认值为 1440（即每天一次）。该变量可在 .env 文件中修改，修改后重启服务生效。检测结果会存储在 Link 模型的 status 字段中，前端界面默认仅标记失效链接并显示警示图标，不会自动隐藏或删除失效链接，管理员可在管理界面中根据状态进行过滤或批量处理。

问题：批量导入链接时支持哪些输入格式？对 URL 格式有何要求？

回答：批量导入功能支持纯文本格式（每行一个 URL）、CSV 格式（包含 url、title、description、category 列）以及 JSON 数组格式。系统会对每个 URL 进行格式校验，要求必须包含协议头（http:// 或 https://）。若导入的 URL 缺少协议头，系统会自动尝试添加 http:// 前缀，但建议用户提前规范数据格式以获得最佳导入效果。
